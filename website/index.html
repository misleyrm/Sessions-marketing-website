Welcome to SESSIONS APP!

Communication is key

Communication is the key to group progress. When we stop communicating with our team members, we miss opportunities to get help or to help out with tasks that are best addressed as a group.

For teams that hold stand-ups

SESSIONS APP is a web application for teams that hold stand-up meetings on a daily or regular basis. SESSIONS APP has two main modules, the team module and the individual user module.

Individual user module

Each individual user can create personal lists and tasks under each list. Users can assign deadlines to tasks, mark tasks as completed, flag tasks as important, or create blockers for specific tasks. Personal lists are visible only to the user that creates them.

Team module

Users can invite other users to collaborate on a project list. Guests can add personal tasks to the project list or can have tasks assigned to them by the owner of the list. Tasks on the project list are visible to all the guests of the list but can only be modified by the creator of the task or the owner of the list.

Work in progress tasks (WIPs)

All tasks that have not been completed are displayed as Work-in-progress tasks. Work-in-progress tasks are also known as WIPs. WIPs can be further defined by assigning them a deadline, an "important" flag, or a blocker. WIPs are displayed in the WIP section of their parent list until marked as completed or until they're deleted from the list.

Completed tasks

Work-in-progress tasks can be marked as completed. Completed tasks are displayed in the completed tasks section of their parent list for one day on the day following completion.

Blockers

Blockers are situations getting in the way of completing a specific task. Creating blockers for tasks lets other members of a project list know that you're having a hard time completing a task.
Blockers can be situations:
I need to learn how to use the new coffee machine to make an afternoon batch.
Or other members:
I need Jane to collect payment from client x before Friday.
If other team members are blockers in a task, they can be tagged on a task blocker using the following syntax:
@name
You can also set up notifications to be reminded when another team member needs your assistance in undoing a blocker.

Notifications & Alerts

Blocker notifications
Users can set up notifications to be reminded when another team member needs their assistance in undoing a blocker.
Alerts
Users can set up notifications to be reminded of an upcoming task deadline.
Group list owners can set up notifications to receive alerts when tasks in their group list are modified.
Notifications and alerts can be received via Slack, e-mail, or text message.

Using SESSIONS APP
during stand-up meetings

SESSIONS APP was created with a single goal in mind: making stand-up meetings more helpful to all team members. All members of a stand-up meeting must be added to a project list. The home page for the project list can be used as a reference by members before, during, or after the stand-up meeting. Tasks completed on the previous day are always displayed in the "Completed" section under each team member's name.

Open tasks are displayed in each member's "Work in progress" section. Tasks can be marked as completed, flagged with a blocker, or unblocked during the stand-up, helping the team move faster while creating a record for the day's stand-up meeting. SESSIONS APP stores historical data and team members can look up completed items, work-in-progress items, and blockers from any past date.
